# AiNiee Word翻译优化 - 解决方案总结

## 📌 问题确认

您提到的问题是:**AiNiee在翻译Word文档时,按照XML中的格式切分太碎,导致翻译质量不好。**

经过深入代码分析,我确认了这个问题的存在和根本原因。

## 🔍 核心问题

### 当前行为

AiNiee当前的处理流程:

1. **读取Word文档** → 解析为XML
2. **查找所有`<w:t>`标签** → 每个标签是一个文本片段
3. **每个`<w:t>`成为一个独立的翻译单元** → 发送给大模型

### 为什么会切得很碎?

Word文档中,**只要文本格式不同(粗体、斜体、颜色、字体等),就会产生新的`<w:t>`标签**。

**示例:**

原始句子:
> 这是一个**重要的**完整句子。

Word XML结构:
```xml
<w:p>
    <w:r><w:t>这是一个</w:t></w:r>
    <w:r><w:rPr><w:b/></w:rPr><w:t>重要的</w:t></w:r>
    <w:r><w:t>完整句子。</w:t></w:r>
</w:p>
```

**AiNiee会切成3个独立条目:**
1. `这是一个`
2. `重要的`
3. `完整句子。`

**问题影响:**
- ❌ 上下文割裂,模型看不到完整句子
- ❌ 翻译质量下降(单个词语脱离语境)
- ❌ 连贯性差
- ❌ 浪费token

## ✅ 解决方案

### 核心思路: 段落级合并

**将同一段落(`<w:p>`)内的所有文本合并成一个翻译单元**

**改进后的流程:**

1. **读取Word文档** → 解析为XML
2. **按段落`<w:p>`遍历** → 合并段落内所有`<w:t>`的文本
3. **保存格式信息** → 记录每个文本片段的样式(粗体、斜体等)
4. **发送完整段落给大模型翻译**
5. **翻译完成后** → 将译文智能分配回原始格式结构
6. **写回Word文档** → 保持格式不变

**改进效果:**

原来:
```
请翻译:
1. 这是一个
2. 重要的
3. 完整句子。
```

改进后:
```
请翻译:
1. 这是一个重要的完整句子。
```

### 技术要点

1. **格式保留:** 在`CacheItem`中添加`word_format_info`字段,存储每个run的格式信息
2. **智能分配:** 翻译后按比例或标点符号将译文切分并映射回原始run结构
3. **向后兼容:** 保留旧的切分模式作为备选,添加配置开关
